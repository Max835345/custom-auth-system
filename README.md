### Custom Authentication & Authorization System (Django)

> Полностью кастомная система аутентификации и авторизации на Django без использования `django.contrib.auth`

### О проекте

Данный проект реализует собственную систему аутентификации и авторизации на базе Django и Django REST Framework.

В проекте **не используется встроенная система аутентификации Django**.  
Реализованы собственные:

- модель пользователя
- логика регистрации и входа
- JWT-аутентификация
- хеширование паролей
- soft delete
- ролевая модель доступа (RBAC)

Проект демонстрирует понимание:
- архитектуры backend-приложений
- принципов безопасности
- работы с JWT
- проектирования систем авторизации

---

## Функциональность

### Пользователи
- Регистрация
- Авторизация
- Обновление профиля
- Мягкое удаление аккаунта
- UUID как primary key

### Безопасность
- Хеширование паролей (SHA-256)
- JWT-токены
- Проверка токена через middleware-логику
- Soft delete вместо физического удаления

### RBAC
- Модель ролей
- Связь пользователь–роль
- Готовая архитектура для расширения прав доступа

---

## Архитектура проекта

### Архитектурные решения:

- Разделение логики по приложениям
- UUID вместо автоинкремента
- Отделение логики хеширования в utils
- Отдельная JWT-утилита
- Soft delete через флаг `is_deleted`

## API Endpoints

### Регистрация

**POST** `/auth/register/`

```json
{
  "first_name": "A",
  "last_name": "B",
  "middle_name": "C",
  "email": "а@example.com",
  "password": "123456",
  "password_repeat": "123456"
}
Post /auth/login/
{
  "email": "a@example.com",
  "password": "123456"
}
Ответ
{
  "access_token": "jwt_token"
}
